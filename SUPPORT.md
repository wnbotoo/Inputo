# Support

Inputo is pre-release software. Support is community best-effort until packaged releases and a maintainer rotation exist.

## Questions

Use GitHub issues for focused questions about building, running, or contributing to Inputo. Please search existing issues first and include your macOS version, Xcode version, and the command or workflow you tried.

## Bugs

Open a bug report with reproduction steps, expected behavior, actual behavior, and redacted logs. Do not include API keys, private prompts, generated confidential text, sensitive screenshots, or local file paths.

## Feature Requests

Open a feature request for product or developer-experience proposals. Features that affect clipboard writes, app activation, provider calls, file access, permissions, screenshots, or window/app metadata need a privacy and consent model.

## Security

Do not open public issues for vulnerabilities. Follow [SECURITY.md](SECURITY.md).

## Privacy

For privacy-sensitive reports or questions, read [PRIVACY.md](PRIVACY.md) first and avoid posting private content publicly.
